RPNet--Robust P-Wave First-Motion Polarity Determination Using Deep Learning

Author: grosphi Date: 八月 11, 2025  22:52:28

tags: deep learning, P-wave polarity

Network

亮点：

  1. Inception module，通过多尺度卷积核并行处理地震波形数据，从而捕捉不同频率范围内的细节信息，另外，allow greater depth while minimizing the number of parameters
  2. self-attention，注意力机制则允许模型动态地关注那些对于极性判断更为关键的时间段或频段，进一步提升预测的准确性
  3. Monte Carlo dropout，量化预测结果的不确定性
  4. RPNet，如何判断unknown（K），通过std是否超过threshold（ex：0.2）；mRPNet（minor version）：直接分三类（U，D，K）

Previous Problems(from Benchmarks)

  1. 需要time-shift很小，也就是说P波拾取要很准（但是在现有的拾取精度0.05s/0.2s内表现就下滑很多），即使有些Network是用了shift0.5s的数据训练（比如CFM，但我记得ROSS也是）
  2. U/D偏好问题：判断为U的概率会明显大于D，这一点在加入K后更加明显
  3. 加入K后的三分类问题的效果远低于只有U、D的二分类问题，对于K的正确分类相对还高一些，但是会出现很多把U、D分成K的现象，time-shift增大后更加明显（这种结果可能来自：overfitting caused by the low-quality labels in the training data, as many of K labeled data are actually U or D but the preference of analyst classify a U or D as a K.）
  4. 但是不加入K的训练数据又迫使分类中确实没有P波的数据（或者说真正分不清的/没有物理意义的数据）被分为U/D

Training Data

常规处理

  1. 使用不同地区的数据集的合集作为训练集
  2. 数据集的扩张：add time-shift（0-0.5s）；flip；由于原本的数据集中U的数量大于D，所以移除了多出来的U使U的数量等于D（但是flip已经可以使U=D了，我不理解为什么要这样）
  3. 仅使用之前研究中和一致意见（分类相同的数据）
  4. 其他常规处理：1Hz高通滤波；截取4s的窗；8：1：1的训练、验证、测试比例

训练过程：这里看来比较常规，一般的神经网络训练过程

Results

  1. 对于time-shift的抗性明显增强
  2. 不再出现U/D的偏好差距过大
  3. 加入K的三分类问题仍旧没有解决，这通过std的设置来处理分类中结果不是很好的数据
